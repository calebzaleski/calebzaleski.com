This is the workflow from start to finish on how I create and manage the tasks and tables from my database!

Step 1: Page Initialization and Multiple Task Lists Setup

When the page loads, the script queries all task list containers using document.querySelectorAll('.todowrapper'). Each container represents a distinct task list (e.g., "website_list", "threedprinting_list").

For each container, the script retrieves its unique identifier via tableDiv.id and invokes initTodoPage(wrapper, tableId) to initialize event listeners and UI elements specific to that list.

This setup ensures multiple task lists can coexist independently, each with its own input, add button, and display area.

Step 2: User Action - Adding a Task

The user types a new task into the input field (<input class="taskInput">) within a specific task list container.

Upon clicking the add button (<button class="addTaskBtn">), the event triggers the addTask(tableId, wrapper) function for the corresponding list.

Step 3: Data Processing and Sanitization

Inside addTask(), the script selects the input element via wrapper.querySelector('.taskInput') to access the user-entered value.

The raw input string is sanitized by sanitizeInput(), which removes or escapes potentially harmful characters such as <, >, and & to prevent XSS attacks and ensure safe backend communication.

The sanitized string is stored as the task content to be sent to the backend.

Step 4: Backend Communication

The sanitized task data is sent via a fetch() POST request to the backend API endpoint (e.g., https://proxy.calebzaleski.com/add-task).

The request body contains JSON with the task list identifier and the task string, for example: {"table": "website_list", "task": "Complete timeline section"}.

The backend responds with a JSON object indicating success or failure, such as {"inserted": true}.

Step 5: UI Refresh and Data Update

If the backend confirms successful insertion, the input field is cleared to prepare for new entries.

The UI is refreshed by invoking refresh(), which calls fetch_all(tableId) to retrieve the latest task list from the backend and update the display dynamically.

This ensures the user sees the most current task list without needing to reload the page.
